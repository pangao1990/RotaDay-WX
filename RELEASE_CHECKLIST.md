# 班妥了微信小程序发布清单

工程代码已按生产配置运行。首次提审前仍需由小程序管理员在微信公众平台完成以下账号级配置；这些配置无法写入源码，也不能由开发者工具代替。

## 1. 小程序后台必配

- 确认 AppID 为 `wx66f9037b33c344da`，名称、头像、简介和服务类目均已审核通过。
- 在“功能 → 客服”中绑定至少一名可接待的客服人员，并用真机验证“设置 → 联系客服”。
- 在“小程序用户隐私保护指引”中如实声明并审核以下用途：
  - 读取剪贴板：仅在用户点击“从剪贴板恢复”时读取其主动复制的班妥了备份 JSON。
  - 写入相册：仅在用户点击“保存片刻海报”时，将现场生成的图片保存到系统相册。
- 隐私指引中说明：排班、团队成员和偏好设置保存在微信本机缓存，不上传至开发者服务器；用户可在“数据备份”中导出或重置。
- 确认管理后台“客服反馈”入口可查看小程序原生意见反馈及用户主动附带的运行日志。

## 2. 开发者工具验收

- 使用正式 AppID 编译，确认“问题”面板为 0。
- 执行“代码质量”扫描并处理阻断项。
- 在不同模拟器和真机确认系统导航标题与右上角胶囊不重叠，页面内容不侵入状态栏。
- 确认排班、统计、设置使用微信 tabBar 切换，切换时页面栈不会被重复重建，底部安全区无遮挡。
- 清缓存后验证首次隐私授权：同意和拒绝两条路径都应正常，拒绝后其他排班功能仍可使用。
- 分别验证个人版、团队版、月度/年度统计、主题、班次循环、备份导入导出。
- 验证原生“联系客服”“意见反馈”，以及恢复备份、保存海报触发的按需授权流程。
- 确认上传包小于微信主包 2 MB 限制，且没有把 `tests/`、README、发布清单和 `package.json` 打入代码包。

## 3. 真机回归

- 至少覆盖一台 iPhone 和一台 Android 微信最新版；若面向鸿蒙微信用户，也需单独验证当前微信版本支持情况。
- 验证跨午夜班次、月份切换、单日覆盖、团队复制粘贴和 CSV 分享。
- 验证相册首次授权、拒绝后“去设置”、再次保存三条路径。
- 验证备份 JSON 可从一台设备导出并在另一台测试设备恢复。
- 断网启动一次，确认核心排班与统计仍可用。

## 4. 提审说明建议

> 班妥了是一款无需登录的本地排班工具。排班、成员与偏好数据默认仅保存在当前微信设备。读取剪贴板仅用于用户主动恢复备份，写入相册仅用于用户主动保存现场生成的片刻海报。客服与意见反馈均使用微信小程序原生能力，不包含后台提醒或持续运行功能。

## 5. 发布流程

1. 在开发者工具中选择“上传”，填写版本号和项目备注。
2. 在微信公众平台选择该开发版本，补充功能页面与隐私说明后提交审核。
3. 审核通过后由管理员点击“发布”。
4. 发布后从微信搜索、扫码和分享卡片各做一次生产环境冒烟测试。

本清单不包含实际“上传 / 提审 / 发布”动作，执行这些动作前应由小程序管理员确认版本号和审核材料。
